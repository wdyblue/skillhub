# Skill 识别规则

## 目录识别

一个文件夹只要包含 `SKILL.md`，就被识别为一个 skill。

示例：

```txt
some-skill/
  SKILL.md
  scripts/
  references/
```

## 名称提取

优先读取 `SKILL.md` 中第一个 Markdown 标题：

```md
# 图片精修
```

如果没有标题，则使用文件夹名。

## 描述提取

优先读取标题之后第一个非空正文行。

如果无法识别，则读取正文中的第一段非空文本，并截取前 140 个字符。

## Hash

使用 `SKILL.md` 完整正文计算 SHA-256。

## 状态

支持状态：

- 正常
- 常用
- 自建
- 待整理
- 待合并
- 疑似重复
- 已归档
- 已废弃
- 路径丢失

## 质量评分

当前版本每项 10 分，总分 100：

- 标题清晰
- 有描述
- 描述足够长
- 有适用场景
- 有输入说明
- 有输出说明
- 有示例
- 有边界限制
- 结构清晰
- 内容长度合理

## 重复风险

v0.1.0 已实现：

- 内容 hash 完全相同：100 分。
- 名称包含 `v1`、`v2`、`new`、`final`、`backup`、`copy`、`old`、`旧版`、`备份`、`最终`：35 分。

后续版本会增加内容相似度和重复组。
